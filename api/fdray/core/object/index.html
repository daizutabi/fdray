    fdray.core.object

    source module fdray.core.object

    3D objects and operations for ray tracing.

    This module provides a collection of 3D object classes for ray tracing with POV-Ray. Objects can be combined using CSG operations (union, intersection, difference) and can be transformed (scale, rotate, translate). Each object can also have attributes like color, texture, or other POV-Ray specific properties.

    The module structure follows these principles:

    1. All objects inherit from the base class Object
    2. CSG operations are implemented as special object classes
    3. Convenience methods are provided for common transformations
    4. String representation of objects matches POV-Ray SDL syntax

    Classes

    • Object — Base class for all 3D objects.

    • Material — Materials define the appearance of objects in the scene.

    • Csg — Base class for Constructive Solid Geometry (CSG) operations.

    • Union — Union of shapes - points inside any of the shapes are inside the union.

    • Intersection — Intersection of shapes - points inside all shapes are inside the intersection.

    • Difference — Difference of shapes - points inside the first shape and outside all others.

    • Merge — Merge of shapes - similar to union but with different surface calculations.

    • Box — A box defined by two corner points.

    • Cuboid — A rectangular prism defined by a center point and dimensions.

    • Cube — A cube defined by a center point and edge length.

    • Cone — A cone or truncated cone between two points with specified radii.

    • Cylinder — A cylinder defined by two points and a radius.

    • Plane — A plane defined by a normal vector and distance from origin.

    • SkySphere — A sky sphere that defines the background color of the scene.

    • Sphere — A sphere defined by a center point and radius.

    • SphereSweep — A sweep of spheres along a path with specified interpolation.

    • Polyline — A polyline (broken line) represented as a linear sphere sweep.

    • Curve — A smooth curve that passes through all specified points.

    • Text — A text object defined by a string and a font.

    • Torus — A torus defined by a center point and a radius.

    source class Curve(centers: Sequence[Point] | NDArray[np.number], radius: float | Sequence[float], *attrs: Any, **kwargs: Any)

    Bases : Polyline

    A smooth curve that passes through all specified points.

    Create a cubic spline sphere sweep that is guaranteed to pass through all given points, including the start and end points. It uses ghost points to ensure proper curve behavior at the endpoints.

    Unlike standard cubic splines which may not pass through the endpoints, this implementation ensures the curve follows all specified points exactly.

    Parameters

    • centers : Sequence[Point] | NDArray[np.number] — Sequence of 3D points or NumPy array with shape (n, 3) where n is the number of points.

    • radius : float | Sequence[float] — Constant radius or sequence of radii for each point.

    • **kwargs : Any — Additional keyword attributes.

    source classmethod Text.set_font(font_spec: str | Path) str | None

    Set the font file for the text object.

    This method determines how to handle the font specification based on its format:

    • If font_spec ends with '.ttf', it's treated as a direct file path.
    • Otherwise, it's treated as a font name to search in system fonts.

    Parameters

    • font_spec : str | Path — Font specification, either a path to a TTF file or a font name. If it's a font name, the method will search for matching fonts in the system fonts directory.

    Returns

    • str — Path to the font file if found, None otherwise.

    Raises

    • ImportError — If matplotlib is not installed and a font name search is attempted.
